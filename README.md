# Trading System 交易系统技能

这是一个面向股票、ETF 等公开市场的低频波段交易规则执行仓库。它的目标不是预测收益，也不是给出买卖建议，而是把候选标的评分、仓位计算、风险检查和交易复盘统一到一套可重复执行的规则里。

## 能力范围

- 候选标的评分：按 100 分模型评估市场环境、趋势结构、买点质量、盈亏比、成交量、流动性、板块强度和组合适配度。
- 单笔交易检查：核对入场价、止损价、第一目标价、R 倍数、市场状态和组合风险。
- 仓位计算：根据账户总资产、单笔最大账户风险和止损幅度计算建议买入金额。
- 持仓复盘：检查当前 R 倍数、止损状态、止盈状态和组合风险影响。
- 交易回放：计算实际 R 结果，区分系统内正常亏损和执行错误。
- 周/月复盘：统计胜率、平均盈利 R、平均亏损 R、期望值、执行率、回撤和熔断状态。

## 重要边界

- 本仓库只提供交易系统规则执行，不构成任何投资建议。
- 评分是规则匹配度，不是上涨概率或收益预测。
- 不会凭空编造实时行情数据；缺少价格、成交量、均线、账户或持仓数据时，应明确补充数据或标注假设。
- 硬性过滤条件优先于评分结果。触发硬过滤时，即使总分较高，也不能按普通候选标的处理。

## 交易系统详解

这套系统的核心思想是：不追求每次判断正确，而是让每一笔交易在买入前就明确“为什么买、错了亏多少、对了怎么处理、什么时候复盘”。它更像一套执行约束，而不是一套预测模型。

系统先判断市场环境，再筛选标的和买点，随后用止损距离反推仓位，最后通过 R 倍数和复盘指标评估执行质量。只要关键输入缺失，系统就应该降低结论确定性，不能把不完整数据包装成明确交易信号。

### 适用风格

| 项目 | 说明 |
|---|---|
| 适用市场 | 股票、ETF 等公开市场标的 |
| 交易周期 | 数天到数周 |
| 交易风格 | 低频、波段、中短线、少盯盘 |
| 决策节奏 | 盘前计划，盘中执行，收盘复盘 |
| 核心目标 | 控制单笔亏损、限制组合回撤、保留趋势利润 |
| 不适合场景 | 高频交易、日内抢单、无止损逻辑的消息交易、无法量化风险的主观交易 |

### 整体执行流程

| 阶段 | 要回答的问题 | 输出 |
|---|---|---|
| 市场判断 | 当前适合承担多少风险？ | 市场评分、总仓位上限 |
| 候选筛选 | 这个标的是否在系统交易范围内？ | 候选清单、排除项 |
| 买点确认 | 是回踩、突破、止跌还是低吸？ | 计划买入价、失效条件 |
| 风险计算 | 如果判断错误，最多亏多少？ | 止损价、1R、建议仓位 |
| 交易计划 | 对了怎么止盈，错了怎么退出？ | 第一目标、第二目标、移动止盈规则 |
| 执行跟踪 | 是否严格按计划执行？ | 持仓状态、当前 R、下一步动作 |
| 周期复盘 | 系统是否有效，错误来自哪里？ | 胜率、平均 R、期望值、执行偏差 |

### 市场环境优先

市场环境决定总仓位，而不是单个标的的故事决定仓位。同样的买点，在强势、震荡和弱势市场里的胜率和容错空间完全不同。

| 市场状态 | 典型特征 | 总仓位参考 |
|---|---|---:|
| 强势 | 指数站上 20 日和 60 日均线，主线持续 3 天以上，赚钱效应扩散 | 50%-80% |
| 震荡 | 指数围绕均线反复，板块轮动快，突破持续性一般 | 30%-50% |
| 弱势 | 指数跌破关键均线，反弹无量，主线退潮，大跌或破位标的增加 | 0%-30% |

市场评分按 0-10 分理解：

| 市场评分 | 状态 | 执行含义 |
|---:|---|---|
| 0-3 | 弱势 | 少做或不做，已有仓位优先降风险 |
| 4-6 | 震荡 | 控制仓位，只做确定性更高的计划内机会 |
| 7-10 | 强势 | 可以提高仓位，但仍必须满足止损和盈亏比要求 |

### 标的选择

系统偏好能解释逻辑、能设置止损、流动性足够、趋势结构没有明显破坏的标的。标的本身再有故事，只要不能量化风险，就不应进入正式交易计划。

| 可接受类型 | 使用方式 |
|---|---|
| 宽基 ETF | 适合稳健配置和低吸，不宜用过高单笔风险 |
| 行业 ETF | 适合跟随持续主线做波段 |
| 龙头股 | 需要流动性、趋势结构和板块强度配合 |
| 趋势股 | 必须严格止损，不能把短线做成长线 |
| 防守类资产 | 用于降低账户整体波动 |

| 应避免类型 | 原因 |
|---|---|
| 长期阴跌标的 | 趋势弱，资金关注度低 |
| 无流动性标的 | 进出困难，滑点不可控 |
| 纯消息刺激标的 | 持续性难验证 |
| 无法设置止损的标的 | 风险无法量化 |
| 完全不了解逻辑的标的 | 容易变成情绪交易 |

### 买点类型

系统只处理计划内买点，常见类型有四类：

| 买点 | 含义 | 使用条件 |
|---|---|---|
| 回踩 | 趋势向上，回踩均线或支撑不破 | 适合波段延续，不追高 |
| 突破 | 放量突破平台、前高或压力位 | 需要成交量确认和失败止损位 |
| 止跌 | 连续调整后缩量企稳，再出现确认信号 | 适合修复行情，不能过早抄底 |
| 低吸 | ETF 或核心资产回落到计划区域 | 更重视仓位控制和分批执行 |

买入前至少要明确：买入逻辑、买入类型、计划买入价、止损价、第一目标价、初始 1R、最坏亏损金额、是否与已有持仓重复。

### 风险和仓位

系统先定义“账户最多愿意亏多少”，再反推“可以买多少”。账户风险不是仓位比例，而是止损触发时账户实际损失的比例。

| 风险档位 | 单笔账户风险 | 单日最大亏损 | 单周最大亏损 | 月度警戒回撤 |
|---|---:|---:|---:|---:|
| 保守 | 0.3%-0.5% | 1% | 2% | 3% |
| 稳健 | 0.5%-1% | 1.5% | 3% | 5% |
| 进取 | 1%-1.5% | 2% | 4% | 8% |

单票仓位还需要叠加上限：

| 类型 | 仓位上限 | 风险建议 |
|---|---:|---:|
| 宽基 ETF | 10%-20% | 0.3%-0.8% |
| 行业 ETF | 5%-15% | 0.5%-1% |
| 龙头股 | 5%-15% | 0.5%-1% |
| 高波动个股 | 3%-8% | 0.3%-0.8% |
| 试错仓 | 2%-5% | 0.2%-0.5% |

如果按止损距离计算出的仓位超过上限，应压回上限；如果计算出的仓位太小，说明止损过宽或买点效率不够。

### 止损规则

止损不是“亏到不舒服再卖”，而是买入前定义的失效条件。常见失效条件包括：

- 跌破关键支撑位。
- 跌破 20 日均线后无法收回。
- 出现放量破位阴线。
- 买入逻辑被证伪。
- 市场风险覆盖个股逻辑。

不同标的的常见止损范围不同：

| 类型 | 常见止损范围 | 原则 |
|---|---:|---|
| 宽基 ETF | 3%-5% | 到计划风险位减仓或退出 |
| 行业 ETF | 5%-8% | 到计划止损位执行 |
| 稳定个股 | 5%-8% | 技术位破坏或风险上限触发时执行 |
| 高波动个股 | 7%-10% | 降低仓位，而不是扩大风险 |
| 试错仓 | 3%-5% | 错了快速退出 |

### 止盈和移动止盈

系统用 R 倍数管理止盈，这样不同价格、不同波动的标的可以用同一种语言复盘。

| 盈利进展 | 操作 |
|---|---|
| +1R | 检查走势是否符合预期，不急于止盈 |
| +1.5R 到 +2R | 可减仓 1/3，或把止损上移到成本附近 |
| +3R | 再减仓 1/3，剩余仓位用移动止盈 |
| +4R 以上 | 只用趋势或移动止盈处理，不主观猜顶 |

弱势或震荡市场中更重视目标位止盈；强势市场中可以给剩余仓位更多趋势空间。若浮盈大幅回撤，尤其是回撤超过浮盈的一半，应按移动止盈规则处理，而不是重新解释交易逻辑。

### 组合限制

系统不只看单笔交易，还要看组合层面的同向风险。

| 风险项 | 限制 |
|---|---:|
| 同板块仓位 | 不超过账户 25% |
| 同逻辑或同主题仓位 | 不超过账户 25% |
| 高波动板块仓位 | 不超过 15%-20% |
| 全部持仓同时止损的理论亏损 | 不超过账户 3%-5% |
| 同板块或同主题理论亏损 | 不超过账户 1.5%-2% |
| 单日新增风险暴露 | 不超过账户 20% |

如果已有持仓和新候选标的是同板块、同逻辑或高度相关资产，新交易必须降低评分或减少仓位，不能只看单个标的的分数。

### 熔断规则

熔断规则用于防止状态失控时继续扩大亏损。

| 触发条件 | 动作 |
|---|---|
| 单日亏损超过 1.5% | 当日停止新开仓 |
| 连续 3 笔亏损 | 暂停新开仓 3 个交易日 |
| 单周亏损超过 3% | 下周总仓位上限降到 30% |
| 月度回撤超过 5% | 总仓位上限降到 20% |
| 月度回撤超过 8% | 暂停真实资金新开仓，至少复盘 20 笔历史交易 |
| 2 次未按计划止损 | 暂停交易，写清原因和纠正规则后再恢复 |

熔断条件优先级高于任何看起来有吸引力的机会。

### 复盘指标

系统复盘不只看赚亏金额，更关注规则是否被执行。

| 指标 | 用途 |
|---|---|
| 胜率 | 判断交易结果分布，但不能单独评价系统 |
| 平均盈利 R | 判断盈利交易能否留住利润 |
| 平均亏损 R | 判断亏损是否被限制在计划内 |
| 期望值 | 综合评估系统是否有正向结果 |
| 最大回撤 | 判断账户波动是否超过承受范围 |
| 执行率 | 判断亏损来自系统本身还是执行偏差 |
| 错误分类 | 找出买点、仓位、止损、止盈、情绪或计划外问题 |

一个系统允许正常亏损，但不允许反复出现同类执行错误。复盘的重点是把错误转化为下一轮更明确的行为约束。

### 禁止行为

- 临时追涨。
- 亏损后情绪化补仓。
- 无计划买入。
- 到止损位后幻想反弹。
- 把短线计划做成长线被套。
- 因为想回本而加大仓位。
- 买入和已有持仓高度重复的标的却不降低风险。
- 事后只看盈亏，不记录是否按计划执行。

## 快速开始

批量评分脚本只依赖 Python 标准库，可直接运行：

```bash
python3 scripts/score_candidates.py candidates.csv
```

输出到 CSV 文件：

```bash
python3 scripts/score_candidates.py candidates.csv -o scored.csv
```

输出 JSON：

```bash
python3 scripts/score_candidates.py candidates.csv --json
```

带账户规模和单笔风险计算建议仓位：

```bash
python3 scripts/score_candidates.py candidates.csv --account 100000 --risk 1 -o scored.csv
```

`--risk` 可以传小数或百分数。`0.01` 和 `1` 都表示 1% 单笔账户风险。

## CSV 字段

推荐输入字段：

```text
symbol,name,price,entry,stop_loss,target1,ma20,ma60,volume,avg_volume_20,
support,resistance,market_score,trend_score,buy_point_score,volume_score,
liquidity_score,sector_score,portfolio_score,sector,current_sector_exposure
```

常用字段说明：

| 字段 | 说明 |
|---|---|
| `symbol` / `name` | 标的代码和名称 |
| `price` / `entry` | 当前价格或计划买入价；`entry` 优先用于风险计算 |
| `stop_loss` | 止损价 |
| `target1` | 第一目标价 |
| `ma20` / `ma60` | 20 日、60 日均线，用于趋势结构推导 |
| `volume` / `avg_volume_20` | 当前成交量与 20 日均量，用于成交量评分 |
| `support` / `resistance` | 支撑位和压力位 |
| `market_score` | 市场评分，0-10 |
| `trend_score` | 趋势分，可直接给分，也可由均线和支撑位推导 |
| `buy_point_score` | 买点质量分 |
| `volume_score` | 成交量分 |
| `liquidity_score` | 流动性分 |
| `sector_score` | 板块或主题强度分 |
| `portfolio_score` | 与当前组合的适配度 |
| `current_sector_exposure` | 当前同板块仓位占比，用于集中度过滤 |

如果直接提供组件分数，脚本会优先使用输入分数；否则会根据价格、均线、成交量、买点类型等字段做保守推导。

## 评分模型

总分为 100 分：

| 维度 | 分值 |
|---|---:|
| 市场环境 | 20 |
| 趋势结构 | 20 |
| 买点质量 | 20 |
| 风险收益比 | 15 |
| 成交量行为 | 10 |
| 流动性 | 5 |
| 板块/主题强度 | 5 |
| 组合适配度 | 5 |

分数等级：

| 分数 | 结论 |
|---:|---|
| 85-100 | high-quality candidate |
| 70-84 | watchlist candidate |
| 60-69 | wait for confirmation |
| 低于 60 | eliminated or low priority |

## 硬性过滤

以下情况会触发硬过滤或限制激进操作：

- 没有有效止损位。
- 第一目标价低于 2R。
- 流动性过低。
- 趋势明显破坏。
- 市场评分低于 4，却仍计划正常或激进仓位。
- 同板块或同逻辑暴露超过系统集中度限制。
- 已触发连续亏损、回撤超限、止损执行失效等熔断条件。

## 仓位计算

核心公式：

```text
stop_loss_pct = abs(entry - stop_loss) / entry
initial_1R = abs(entry - stop_loss)
target_r = abs(first_target - entry) / initial_1R
buy_amount = account_equity * max_account_risk_per_trade / stop_loss_pct
worst_case_loss = buy_amount * stop_loss_pct
position_pct = buy_amount / account_equity
```

如果止损幅度为 0 或无法计算，交易无效。计算出的仓位还需要再套用单票仓位、同方向仓位、组合理论亏损和熔断规则。

## 输出字段

批量评分脚本会输出：

- `total_score`：总分
- `grade`：系统结论
- `hard_filters`：触发的硬过滤
- 各维度分数：`market_points`、`trend_points`、`buy_point_points`、`risk_reward_points` 等
- `initial_1r`：初始 1R
- `target_r`：第一目标价对应 R 倍数
- `suggested_buy_amount`：建议买入金额，仅在提供 `--account` 和 `--risk` 时计算
- `position_pct`：建议仓位占账户比例
- `worst_case_loss`：最坏情况下亏损金额
