# 通用交易系统构建手册

> 适用范围：股票、ETF等公开市场交易。  
> 适合风格：波段交易、中短线交易、低频交易、少盯盘交易。  
> 核心目标：建立一套可重复、可执行、可复盘的交易流程，让交易从“凭感觉”变成“按系统”。

> 重要说明：本手册用于建立交易流程和风险控制框架，不构成任何具体投资建议。任何规则都应先经过历史样本验证和小仓试运行，再进入正式资金执行。

---

## 1. 为什么需要交易系统

交易系统的目的不是让每一笔交易都赚钱，而是让交易行为变得稳定、可控、可复盘。

一个成熟的交易系统需要解决六个问题：

| 模块 | 解决的问题 |
|---|---|
| 交易范围 | 买什么，不买什么 |
| 入场规则 | 什么时候可以买 |
| 仓位管理 | 买多少 |
| 止损规则 | 错了怎么办 |
| 止盈规则 | 对了怎么办 |
| 复盘机制 | 如何持续改进 |

没有交易系统时，常见问题包括：

1. 看到上涨就追，看到下跌就怕。
2. 买入前没有计划，买入后靠情绪。
3. 盈利时舍不得卖，亏损时不愿认错。
4. 仓位忽大忽小，风险无法控制。
5. 同一类资产买得过多，账户波动过大。
6. 事后只记得结果，不知道自己错在哪里。

交易系统的意义在于：

> 不追求每次判断正确，而是确保判断错误时损失有限，判断正确时利润能够留下。

### 1.1 系统使用边界

本系统适合低频、波段、中短线交易，不适合所有交易场景。

| 适合使用 | 不适合使用 |
|---|---|
| 股票、ETF、基金等可设置止损的公开市场标的 | 高频交易、日内打板、盘口秒级交易 |
| 数天到数周的持仓周期 | 期权、杠杆合约、重杠杆期货 |
| 盘前计划、盘中执行、盘后复盘 | 纯消息博弈、内幕消息、无法验证逻辑的交易 |
| 低频、少盯盘、纪律型交易者 | 不愿止损、无法记录交易、频繁冲动交易者 |

如果一个标的无法设置有效止损、流动性不足、交易逻辑无法复盘，应直接排除。

## 2. 交易系统的基本原则

### 2.1 先控制风险，再追求收益

任何交易都应该先问：

- 最坏会亏多少？
- 这个亏损是否能接受？
- 如果连续亏几次，账户是否还能正常运转？

只有风险可控的交易，才值得参与。

### 2.2 先有退出计划，再考虑买入

买入前必须知道：

- 为什么买？
- 买错了在哪里卖？
- 买对了在哪里减？
- 如果走势不符合预期，如何处理？

没有退出计划的买入，本质上不是交易，而是下注。

### 2.3 仓位比判断更重要

一次判断错误不可怕，重仓判断错误才可怕。

仓位管理的核心不是为了赚最多，而是为了避免一次错误伤到账户根基。

### 2.4 系统要简单

越复杂的系统，越难执行。

好的交易系统应该满足：

- 规则清楚
- 信号明确
- 操作简单
- 容易复盘
- 不依赖临场情绪

### 2.5 执行比预测更重要

市场不可控，执行可控。

交易者真正能控制的只有：

- 是否买入
- 买多少
- 何时止损
- 何时止盈
- 是否复盘

## 3. 确定自己的交易定位

构建交易系统前，先明确自己的交易定位。

| 问题 | 可选答案 |
|---|---|
| 交易市场 | 股票、ETF、基金、期货、外汇、加密资产 |
| 交易周期 | 日内、短线、波段、中线、长线 |
| 盯盘时间 | 全天盯盘、盘中偶尔看、只看收盘 |
| 风险承受 | 保守、稳健、进取 |
| 资金目标 | 稳定增值、跑赢指数、阶段性收益 |
| 最大回撤 | 5%、10%、15%、20%以上 |

### 示例定位

对于大多数普通投资者，更适合的定位是：

| 项目 | 建议 |
|---|---|
| 交易周期 | 波段 / 中短线 |
| 持仓时间 | 数天到数周 |
| 盯盘方式 | 盘前计划 + 盘中执行 + 收盘复盘 |
| 总仓位 | 根据市场环境控制 |
| 风险控制 | 单笔亏损有限，账户回撤可控 |

### 3.1 个人风险参数

交易系统必须先定义个人风险上限，再定义买点。

| 参数 | 保守型 | 稳健型 | 进取型 |
|---|---:|---:|---:|
| 单笔最大账户风险 | 0.3%—0.5% | 0.5%—1% | 1%—1.5% |
| 单日最大亏损 | 1% | 1.5% | 2% |
| 单周最大亏损 | 2% | 3% | 4% |
| 月度警戒回撤 | 3% | 5% | 8% |
| 月度强制降仓回撤 | 5% | 8% | 10% |

账户风险指的是“如果止损被触发，账户总资产最多亏多少”，不是买入仓位比例。

---

## 4. 交易范围设计

交易系统必须规定“能买什么”和“不能买什么”。

### 4.1 可交易范围

| 类型 | 特点 |
|---|---|
| 宽基ETF | 分散、波动相对较低，适合稳健配置 |
| 行业ETF | 跟随行业趋势，适合波段交易 |
| 龙头股 | 流动性较好，基本面相对清晰 |
| 趋势股 | 适合顺势交易，但要严格止损 |
| 防守类资产 | 用于降低账户波动 |

### 4.2 不建议参与的类型

| 类型 | 原因 |
|---|---|
| 长期阴跌标的 | 趋势弱，资金关注度低 |
| 无流动性标的 | 买卖困难，滑点较大 |
| 纯消息刺激标的 | 持续性不确定 |
| 无法设置止损的标的 | 风险无法量化 |
| 完全不了解逻辑的标的 | 容易被情绪左右 |

## 5. 市场环境判断

市场环境决定仓位。

同样的买点，在强势市场和弱势市场中的胜率完全不同。

### 5.1 市场状态分类

| 市场状态 | 可执行定义 | 仓位策略 |
|---|---|---|
| 强势市场 | 指数收盘在20日线、60日线上方；20日线向上；主线板块持续3个交易日以上；上涨家数和放量情况支持 | 50%—80% |
| 震荡市场 | 指数在20日线、60日线附近反复；板块轮动快；突破持续性差；赚钱效应不稳定 | 30%—50% |
| 弱势市场 | 指数收盘跌破20日线或60日线；反弹无量；主线退潮；跌停或大跌数量增加 | 0%—30% |

市场状态以收盘结果为主要依据，盘中波动只作为辅助观察，不用于频繁切换仓位。

### 5.2 判断市场环境的参考指标

| 指标 | 观察重点 |
|---|---|
| 指数位置 | 是否站上20日线、60日线 |
| 成交量 | 是否放量上涨或缩量下跌 |
| 主线持续性 | 热点能否持续3天以上 |
| 涨跌家数 | 市场整体赚钱效应 |
| 龙头表现 | 强势股是否继续走强 |
| 跌停 / 大跌数量 | 风险偏好是否下降 |

### 5.3 仓位调整原则

| 情况 | 操作 |
|---|---|
| 市场强势，主线明确 | 可以提高仓位 |
| 市场震荡，轮动较快 | 控制仓位，快进快出 |
| 市场弱势，主线退潮 | 降低仓位，少做或不做 |
| 连续亏损 | 降仓复盘 |
| 看不懂市场 | 保持现金 |

### 5.4 市场环境评分表

每天收盘后给市场打分，避免只凭感觉判断强弱。

| 项目 | 0分 | 1分 | 2分 |
|---|---|---|---|
| 指数趋势 | 跌破20日线和60日线 | 20日线附近震荡 | 站上20日线和60日线 |
| 成交量 | 下跌放量或上涨缩量 | 量能无明显方向 | 上涨放量或回调缩量 |
| 主线持续性 | 无主线或主线退潮 | 多方向快速轮动 | 主线持续3天以上 |
| 赚钱效应 | 大跌和破位标的多 | 分化明显 | 强势标的扩散 |
| 龙头表现 | 龙头补跌或破位 | 龙头震荡 | 龙头继续新高或抗跌 |

| 总分 | 市场状态 | 总仓位上限 |
|---:|---|---:|
| 0—3 | 弱势 | 0%—30% |
| 4—6 | 震荡 | 30%—50% |
| 7—10 | 强势 | 50%—80% |

## 6. 选股与选ETF规则

### 6.1 选择标的的基本条件

一个标的至少应满足以下条件中的三项：

1. 所属方向有明确逻辑。
2. 趋势没有明显破位。
3. 成交量配合良好。
4. 价格接近有效支撑或突破关键压力。
5. 所在板块有资金关注。
6. 盈亏比合理。
7. 能清楚设置止损位。

### 6.2 优先选择

| 类型 | 说明 |
|---|---|
| 趋势向上的标的 | 顺势交易胜率更高 |
| 回调不破关键支撑的标的 | 风险相对可控 |
| 放量突破平台的标的 | 资金行为较明确 |
| 行业龙头或核心ETF | 流动性和识别度较高 |
| 逻辑清晰的主线方向 | 更容易跟踪和复盘 |

### 6.3 谨慎选择

| 类型 | 风险 |
|---|---|
| 连续大涨后的标的 | 追高风险大 |
| 趋势破位后的标的 | 可能进入下跌趋势 |
| 低位长期横盘无量标的 | 资金关注不足 |
| 反复冲高回落标的 | 抛压较重 |
| 与已有持仓高度重复的标的 | 增加集中风险 |

### 6.4 标的筛选顺序

每个候选标的按同一顺序筛选，任何一关不通过就不进入交易计划。

| 顺序 | 筛选项 | 通过标准 |
|---:|---|---|
| 1 | 市场环境 | 市场评分不低于4分；弱势市场只允许低仓位或不交易 |
| 2 | 板块逻辑 | 所属方向有明确催化、趋势或资金关注，不只依赖单日消息 |
| 3 | 标的趋势 | 价格未明显跌破20日线或关键支撑，结构没有连续破位 |
| 4 | 流动性 | 成交活跃，买卖价差可接受，能正常进出 |
| 5 | 买点位置 | 回踩、突破、止跌、低吸四类买点之一 |
| 6 | 止损设置 | 能明确写出判断错误的位置 |
| 7 | 盈亏比 | 第一目标至少达到2R，或有清晰的移动止盈空间 |
| 8 | 组合风险 | 与已有持仓不是同一方向过度重复 |

### 6.5 关键术语的量化定义

| 术语 | 执行定义 |
|---|---|
| 放量 | 当日成交量大于近20日平均成交量的1.3倍；突破交易优先使用1.5倍 |
| 缩量 | 当日成交量低于近20日平均成交量的0.8倍 |
| 有效突破 | 收盘价站上关键压力位，且成交量放大，次日不快速跌回压力位下方 |
| 有效跌破 | 收盘价跌破关键支撑位，或盘中跌破后收盘无法收回 |
| 关键支撑 | 前低、平台下沿、20日线、60日线、趋势线或放量启动位置 |
| 主线持续 | 同一方向连续3个交易日以上保持强度，且核心标的没有明显补跌 |

## 7. 入场规则

入场规则要明确，不能因为“感觉要涨”就买。

### 7.1 常见入场类型

| 类型 | 条件 | 适合场景 |
|---|---|---|
| 回踩买入 | 趋势向上，回踩重要均线或支撑不破 | 波段交易 |
| 突破买入 | 放量突破平台、前高或压力位 | 主线行情 |
| 止跌买入 | 连续调整后缩量企稳，出现放量阳线 | 修复行情 |
| 分批低吸 | 大方向未破，回调到计划区域 | ETF或稳健标的 |

### 7.2 入场前检查

每次买入前，必须回答：

| 问题 | 说明 |
|---|---|
| 为什么买？ | 交易逻辑是否清楚 |
| 买点是什么？ | 回踩、突破、止跌还是低吸 |
| 止损在哪里？ | 跌破哪里说明判断错误 |
| 目标在哪里？ | 第一止盈位在哪里 |
| 买多少？ | 仓位是否符合规则 |
| 盈亏比是否合理？ | 预期收益是否大于潜在亏损 |
| 是否和已有持仓重复？ | 避免同一方向过重 |

### 7.3 盈亏比要求

一笔交易至少应满足：

> 预期收益 : 潜在亏损 ≥ 2 : 1

例如：

| 项目 | 数值 |
|---|---:|
| 买入价 | 10元 |
| 止损价 | 9.5元 |
| 潜在亏损 | 5% |
| 第一目标价 | 11元 |
| 预期收益 | 10% |
| 盈亏比 | 2 : 1 |

如果止损空间是5%，但预期上涨空间只有5%，这笔交易并不划算。

### 7.4 R倍数规则

把每笔交易的风险定义为1R。

| 项目 | 说明 |
|---|---|
| 1R | 买入价到止损价之间的亏损幅度或亏损金额 |
| 2R | 预期收益达到潜在亏损的2倍 |
| -1R | 按计划止损 |
| +1R | 盈利等于初始风险 |
| +2R | 达到最低合格盈亏比 |

示例：买入价10元，止损价9.5元，1R为0.5元。如果第一目标价为11元，则预期盈利1元，等于2R。

每笔交易在买入前必须写出：初始1R是多少、第一止盈位是多少R、最坏亏损金额是多少。

## 8. 仓位管理系统

仓位管理决定账户能不能长期活下来。

### 8.1 总仓位规则

| 市场状态 | 总仓位 |
|---|---:|
| 强势市场 | 50%—80% |
| 震荡市场 | 30%—50% |
| 弱势市场 | 0%—30% |
| 连续亏损阶段 | 0%—30% |
| 看不懂市场 | 0%—20% |

### 8.2 单笔仓位规则

单笔仓位不能只按“想买多少”决定，必须由账户风险和止损距离共同决定。

> 单笔买入金额 = 账户总资产 × 单笔最大账户风险 ÷ 止损幅度

例如账户100万元，单笔最大账户风险为1%，止损幅度为5%：

> 单笔买入金额 = 100万 × 1% ÷ 5% = 20万元

| 标的类型 | 单只仓位上限 | 单笔账户风险建议 |
|---|---:|---:|
| 宽基ETF | 10%—20% | 0.3%—0.8% |
| 行业ETF | 5%—15% | 0.5%—1% |
| 龙头股 | 5%—15% | 0.5%—1% |
| 高波动个股 | 3%—8% | 0.3%—0.8% |
| 试错仓 | 2%—5% | 0.2%—0.5% |

计算出的仓位超过单只仓位上限时，按上限执行；计算出的仓位过小，说明止损距离太大或交易性价比不足。

### 8.3 分批建仓规则

| 阶段 | 仓位 | 条件 |
|---|---:|---|
| 试仓 | 计划仓位30% | 出现初步买点 |
| 加仓 | 计划仓位30% | 买入后走势符合预期 |
| 完成建仓 | 计划仓位40% | 突破确认或回踩不破 |

### 8.4 加仓原则

允许加仓的情况：

1. 买入后走势符合预期。
2. 标的突破关键压力。
3. 回踩不破支撑。
4. 市场环境支持。
5. 加仓后总风险仍可控。

禁止加仓的情况：

1. 买入后亏损，仅为了摊低成本。
2. 跌破止损位后仍想补仓。
3. 市场环境转弱。
4. 单一方向已经过重。
5. 没有新的确认信号。

### 8.5 行业集中度控制

| 情况 | 规则 |
|---|---|
| 同一行业 | 不超过总资产25% |
| 同一逻辑 | 不超过总资产25% |
| 高波动行业 | 不超过总资产15%—20% |
| 防守类资产 | 可适当提高，但仍需设规则 |

### 8.6 组合风险限制

| 风险项 | 上限 |
|---|---:|
| 所有未触发止损时的合计理论亏损 | 不超过账户总资产3%—5% |
| 同一行业或同一逻辑最大风险敞口 | 不超过账户总资产1.5%—2% |
| 单日新增仓位 | 不超过账户总资产20% |
| 连续亏损后的新增仓位 | 不超过正常仓位的一半 |

持仓数量越多，越要看“同时止损会亏多少”，而不是只看每只仓位是否合理。

## 9. 止损系统

止损是交易系统的生命线。

止损不是承认失败，而是承认市场没有按计划走。

### 9.1 固定比例止损

固定比例止损用于快速约束风险，但最终应结合技术位和波动率。

| 标的类型 | 常用止损区间 | 操作 |
|---|---:|---|
| 宽基ETF | -3%—-5% | 跌破计划位减仓或止损 |
| 行业ETF | -5%—-8% | 到计划止损位执行 |
| 稳健个股 | -5%—-8% | 跌破技术位或达到上限处理 |
| 高波动个股 | -7%—-10% | 必须降低仓位，不能只放宽止损 |
| 试错仓 | -3%—-5% | 错了快速退出 |

### 9.2 技术位止损

| 情况 | 操作 |
|---|---|
| 跌破关键支撑 | 减仓或止损 |
| 跌破20日线且无法收回 | 波段仓降低 |
| 放量长阴破位 | 立即处理 |
| 反弹无量、继续走弱 | 不再幻想 |
| 买入逻辑失效 | 直接退出 |

### 9.2.1 波动率止损

如果标的波动较大，可用ATR辅助设置止损。

| 方法 | 规则 |
|---|---|
| ATR止损 | 止损距离可设为1.5—2倍ATR |
| 技术位优先 | 技术位止损和ATR止损冲突时，选择更能代表交易逻辑失效的位置 |
| 仓位调整 | 止损距离变大时，仓位必须按风险公式同步降低 |

不能因为标的波动大就随意扩大亏损上限。

### 9.3 时间止损

有些交易虽然不亏，但迟迟不涨，也会占用资金和注意力。

| 情况 | 操作 |
|---|---|
| 买入后3—5天无明显反应 | 重新评估 |
| 买入后超过计划周期仍不涨 | 减仓或退出 |
| 市场主线切换 | 退出低效持仓 |

### 9.4 止损纪律

必须避免：

1. 临时取消止损。
2. 跌破止损后找理由继续持有。
3. 亏损后补仓摊平。
4. 把短线交易改成长线持有。
5. 用基本面理由掩盖交易纪律失效。

### 9.5 跳空与极端情况处理

| 情况 | 操作 |
|---|---|
| 开盘跳空跌破止损位 | 不等待反弹，先按计划降低风险 |
| 重大利空导致流动性不足 | 能成交时优先处理风险，复盘滑点损失 |
| 系统性暴跌 | 个股逻辑服从市场风险，总仓位优先下降 |
| 停牌或无法卖出 | 记录为不可控风险，后续降低类似标的仓位上限 |

## 10. 止盈系统

止盈的目的是让利润真正留在账户里。

### 10.1 分批止盈

| 浮盈幅度 | 操作 |
|---|---|
| +5%—6% | 抬高止损到成本附近 |
| +8%—10% | 卖出1/3 |
| +12%—15% | 再卖出1/3 |
| +20%以上 | 剩余仓位用趋势持有 |

### 10.2 移动止盈

移动止盈适合趋势行情。

| 情况 | 操作 |
|---|---|
| 从最高浮盈回撤1/3 | 减仓 |
| 从最高浮盈回撤1/2 | 至少卖出一半 |
| 跌破10日线 | 短线仓减 |
| 跌破20日线 | 波段仓减 |
| 放量跌破平台 | 退出 |

### 10.3 目标位止盈

适用于震荡行情或压力位明显的标的。

| 情况 | 操作 |
|---|---|
| 到达前高压力 | 减仓 |
| 到达平台上沿 | 减仓 |
| 放量滞涨 | 减仓 |
| 冲高回落 | 减仓 |
| 连续加速上涨 | 分批落袋 |

### 10.4 止盈纪律

盈利后最容易犯的错误：

1. 觉得还能涨，不愿卖。
2. 计划止盈位到了，却临时提高目标。
3. 盈利回撤后仍然幻想。
4. 没有移动止盈，利润坐电梯。
5. 把短线盈利变成长期套牢。

止盈不是卖飞，而是完成计划。

### 10.5 按R倍数止盈

用R倍数管理止盈，可以让不同价格、不同波动的标的用同一套语言复盘。

| 盈利进展 | 操作 |
|---|---|
| +1R | 检查走势是否符合预期，不急于止盈 |
| +1.5R—+2R | 可减仓1/3，或把止损上移到成本附近 |
| +3R | 再减仓1/3，剩余仓位用移动止盈 |
| +4R以上 | 只用趋势或移动止盈处理，不再主观猜顶 |

如果市场处于弱势或震荡状态，可以更重视目标位止盈；如果市场处于强势状态，可以给剩余仓位更多趋势空间。

### 12.2 盘中流程

盘中只执行计划。

允许做：

1. 到止损位，执行。
2. 到止盈位，执行。
3. 符合买点，按计划买入。
4. 异常放量破位，降低风险。

不允许做：

1. 临时追涨。
2. 情绪化补仓。
3. 因为恐慌乱卖。
4. 因为想回本临时加仓。
5. 买计划外标的。

## 15. 每周复盘模板

每周至少复盘一次。

| 项目 | 问题 |
|---|---|
| 本周账户收益 | 盈利还是亏损 |
| 最大盈利标的 | 为什么赚 |
| 最大亏损标的 | 为什么亏 |
| 是否严格止损 | 有没有犹豫 |
| 是否严格止盈 | 有没有幻想 |
| 仓位是否合理 | 是否过重 |
| 是否买了计划外标的 | 为什么买 |
| 是否有情绪交易 | 贪婪、恐惧、想回本 |
| 本周总R结果 | 所有交易结果R合计 |
| 本周平均R | 平均每笔赚/亏多少R |
| 最大执行偏差 | 哪条规则最常被破坏 |
| 下周市场判断 | 强势 / 震荡 / 弱势 |
| 下周总仓位计划 | 多少仓位合适 |

### 每周重点问题

1. 本周有没有违反系统？
2. 哪一笔交易最不该做？
3. 哪一笔交易执行最好？
4. 下周最需要避免什么？
5. 当前系统是否需要微调？

### 15.1 错误分类

复盘时不要只写“亏了”或“市场不好”，必须给每笔问题交易归类。

| 错误类型 | 说明 |
|---|---|
| A类：计划外交易 | 买入前没有写计划，或买了候选池之外的标的 |
| B类：买点错误 | 追高、过早低吸、突破未确认、回踩未企稳 |
| C类：仓位错误 | 单笔风险超标、同方向过重、弱势市场重仓 |
| D类：止损错误 | 到止损位不执行，或临时下移止损 |
| E类：止盈错误 | 到计划位不减仓，或利润大幅回撤不处理 |
| F类：逻辑错误 | 买入理由本身不成立，或误判市场环境 |
| G类：执行良好但正常亏损 | 按系统执行，亏损在计划内 |

G类亏损不需要惩罚系统；A到F类错误必须减少重复发生。

### 月度复盘重点

比起收益率，更应该关注：

1. 是否控制住回撤。
2. 是否严格执行止损。
3. 是否让盈利留下。
4. 是否避免了重仓单一方向。
5. 是否减少了情绪交易。

### 16.1 核心统计指标

| 指标 | 计算方式 | 作用 |
|---|---|---|
| 胜率 | 盈利笔数 / 总交易笔数 | 判断信号质量 |
| 平均盈利R | 盈利交易R总和 / 盈利笔数 | 判断盈利是否拿得住 |
| 平均亏损R | 亏损交易R总和 / 亏损笔数 | 判断止损是否受控 |
| 期望值 | 胜率 × 平均盈利R - 亏损率 × 平均亏损R | 判断系统长期是否有正期望 |
| 最大回撤 | 账户高点到低点的最大跌幅 | 判断风险是否可承受 |
| 计划执行率 | 按计划执行笔数 / 总交易笔数 | 判断问题在系统还是执行 |

如果期望值为负，先停止扩大资金；如果计划执行率低于80%，先修执行纪律，不急着优化买点。

## 17. 情绪管理规则

交易中最大的敌人通常不是市场，而是情绪。

### 17.1 常见情绪陷阱

| 情绪 | 表现 | 风险 |
|---|---|---|
| 贪婪 | 盈利后不愿止盈 | 利润回吐 |
| 恐惧 | 正常波动中乱卖 | 错失趋势 |
| 不甘心 | 亏损后死扛 | 小亏变大亏 |
| 幻想 | 破位后期待反转 | 越套越深 |
| 报复心理 | 亏损后急着赚回来 | 连续犯错 |
| 过度自信 | 连续盈利后加大仓位 | 一次大亏 |

### 17.2 情绪熔断机制

出现以下情况，暂停买入：

1. 单日亏损超过账户1.5%。
2. 连续三笔交易亏损。
3. 连续两次没有执行止损。
4. 因为想回本而想买入。
5. 看到账户后明显焦虑。
6. 盘中频繁改计划。

暂停后只允许：

- 减仓
- 观察
- 复盘
- 等待下一次计划内机会

### 17.3 账户回撤熔断机制

| 触发条件 | 处理方式 |
|---|---|
| 单日亏损超过账户1.5% | 当天停止新开仓，只允许减仓和复盘 |
| 连续3笔交易亏损 | 暂停新开仓3个交易日，复查市场环境和买点质量 |
| 周度亏损超过账户3% | 下周总仓位上限降至30% |
| 月度回撤超过5% | 总仓位上限降至20%，只做最高质量机会 |
| 月度回撤超过8% | 暂停实盘新开仓，至少复盘20笔历史交易后恢复 |
| 连续2次未执行止损 | 暂停交易，直到写出原因和改进措施 |

熔断不是惩罚，而是防止交易者在状态差、市场差或系统失效时继续放大错误。

## 19. 通用交易系统执行版

### 19.1 系统框架

| 模块 | 规则 |
|---|---|
| 交易范围 | 只做自己熟悉、能看懂、能设止损的标的 |
| 市场环境 | 每日按评分表判断，强势提高仓位，弱势降低仓位 |
| 买入方式 | 回踩、突破、止跌、低吸四类计划买点 |
| 建仓方式 | 分批建仓，不一次打满 |
| 加仓原则 | 只在盈利或走势确认后加仓 |
| 止损原则 | 跌破计划位或亏损达到上限就处理 |
| 止盈原则 | 分批止盈 + 移动止盈 |
| 仓位原则 | 用账户风险和止损距离计算仓位，单票有限，同方向有限 |
| 复盘原则 | 每日记录，每周总结，每月统计R倍数和执行偏差 |
| 熔断原则 | 连续亏损、回撤超限、止损失效时暂停或降仓 |

### 19.2 简化执行规则

| 场景 | 操作 |
|---|---|
| 买入前 | 写清楚买点、止损、止盈、1R、仓位、最坏亏损金额 |
| 市场评分0—3分 | 总仓位0%—30%，原则上不主动进攻 |
| 市场评分4—6分 | 总仓位30%—50%，降低交易频率 |
| 市场评分7—10分 | 总仓位50%—80%，只做计划内机会 |
| 买入后亏损达到-1R | 按计划止损或减仓，不下移止损 |
| 盈利达到+1.5R—+2R | 可减仓1/3，或把止损上移到成本附近 |
| 盈利达到+3R | 再减仓1/3，剩余仓位用移动止盈 |
| 浮盈回撤一半 | 必须减仓或退出 |
| 连续3笔亏损 | 暂停新开仓3个交易日 |
| 看不懂市场 | 保持现金 |
| 情绪失控 | 停止交易 |

## 20. 系统优化原则

交易系统不是一成不变的，但也不能频繁乱改。

### 20.1 可以优化的内容

| 内容 | 优化方式 |
|---|---|
| 止损比例 | 根据波动率调整 |
| 止盈方式 | 根据市场强弱调整 |
| 仓位上限 | 根据回撤承受能力调整 |
| 选股条件 | 根据复盘结果优化 |
| 交易周期 | 根据个人时间和性格调整 |

### 20.2 不应频繁改变的内容

| 内容 | 原因 |
|---|---|
| 风险上限 | 是账户安全底线 |
| 止损纪律 | 一旦松动容易失控 |
| 仓位纪律 | 防止大亏 |
| 复盘习惯 | 是系统进化基础 |
| 不情绪交易 | 是长期生存原则 |

### 20.3 判断系统是否有效

重点看以下指标：

1. 最大回撤是否可控。
2. 是否能避免大亏。
3. 盈利交易是否能留下利润。
4. 亏损交易是否能及时退出。
5. 执行后情绪是否更稳定。
6. 长期结果是否比随意交易更好。

### 20.4 实盘前验证流程

任何交易系统都不能只靠读起来合理，必须先验证。

| 阶段 | 要求 | 通过标准 |
|---|---|---|
| 历史样本回放 | 选择30—50笔符合条件的历史交易样本 | 记录每笔计划、止损、止盈和结果R |
| 小仓试运行 | 用正常计划仓位的20%—30%运行1—3个月 | 最大回撤在预设范围内，执行率不低于80% |
| 正式执行 | 逐步恢复正常仓位 | 期望值为正，且没有连续破坏核心纪律 |

如果验证阶段出现大幅回撤、计划执行率低、频繁计划外交易，应先修正规则或降低仓位，不进入正式执行。

## 21. 一句话原则

1. 买入前先想好卖出。
2. 仓位永远比观点重要。
3. 亏损时不要补仓救错。
4. 盈利时不要让利润坐电梯。
5. 趋势破了就按破位处理。
6. 看不懂时现金也是仓位。
7. 连续犯错时先停手。
8. 系统不是为了完美，而是为了可控。
9. 交易不是证明自己正确，而是管理风险。
10. 长期赚钱靠纪律，不靠幻想。

## 22. 最终版本：通用波段交易系统

| 项目 | 规则 |
|---|---|
| 交易周期 | 数天到数周 |
| 市场评分 | 0—3弱势，4—6震荡，7—10强势 |
| 总仓位 | 强势50%—80%，震荡30%—50%，弱势0%—30% |
| 单笔风险 | 单笔最大账户风险通常控制在0.5%—1% |
| 仓位计算 | 账户总资产 × 单笔最大账户风险 ÷ 止损幅度 |
| 单票仓位 | 3%—15%，宽基ETF可放宽到20% |
| 同方向仓位 | 不超过总资产25% |
| 组合理论亏损 | 所有持仓同时止损时，合计亏损不超过账户3%—5% |
| 买入方式 | 计划内回踩、突破、止跌、低吸 |
| 建仓方式 | 分批建仓 |
| 加仓规则 | 只在盈利或走势确认后加仓 |
| 止损规则 | 到-1R、技术位破位或买入逻辑失效时执行 |
| 止盈规则 | +1.5R—+2R可减1/3，+3R再减1/3 |
| 移动止盈 | 浮盈回撤一半必须处理 |
| 复盘要求 | 每日记录，每周复盘，每月统计胜率、平均R、期望值、执行率 |
| 熔断规则 | 连续亏损、回撤超限、止损失效时暂停新开仓 |
| 禁止行为 | 情绪追涨、亏损补仓、无计划买入、破位幻想 |

## 23. 开始执行

从下一笔交易开始，只做四件事：

1. **买前写计划**
   - 买点、仓位、止损、止盈、1R、最坏亏损金额。

2. **盘中按计划执行**
   - 不临时追涨，不临时补仓，不随意取消止损。

3. **收盘后复盘**
   - 记录是否按计划执行，统计结果R，找出偏差，持续修正。

4. **每月验证系统**
   - 统计胜率、平均盈利R、平均亏损R、期望值、最大回撤和执行率。

> 一套交易系统的价值，不在于让你永远不亏，而在于让你每一次亏损都在可承受范围内，让每一次盈利都有机会真正留下。
